# being-language-weaver

とりあえずセットアップのイメージを。

## 全体像

システム全体はfront-BFF-gateway-adapterの構成になっていて、このリポジトリはfront。
BFFは未作成で、gatewayの仕様を参考に、front側で契約を作成する。
openapi.jsonを自動で作成できる仕組みにして、BFF側で取得して、BFFも実装できるようにしたい。

システム全体の仕様については下記のnpmで最新を取得できるようpackage.jsonを設定したい。
npm i @tachiiri-library/specifications

フロントの公開はcloudflare pagesを想定していて、こちらを使ってほしい。
BFFとはservice bindingでつなぐつもり。

トークン消費を考慮して、Codexで詳細な実装を行うつもり。
ただ環境構築周りはclaude codeで行う方が効率が良いので、基本的な動く形を担保したい。

## ツールの目的

現行、自分は下記のような世界観を持っている。

**関係性の定義**
全ては「存在」である
「存在」は「知覚」できる
「知覚」は「開示」を起こす
「開示」は「投企」を生む
「投企」は「自己同一性」を生む
「自己同一性」は「世界観」を生む
「世界観」は「言語」で伝えられる
「言語」が同じ人たちで「世界」ができる

**名詞の定義**
「存在」とは全てである
「知覚」とは存在を感じる事である
「存在者」とは「知覚」を与える単位である
「開示」とは「存在」を「存在者」に変える事である
「投企」とは「自己」と「存在者」の関係を望ましいものに変えようとする意志である
「自己同一性」とは「投企」を抽象化したパターンである
「世界観」とは「存在者」ではない「存在」への「投企」を「自己同一性」によって定義したものである
「言語」とは「世界観」を他の「自己」と共有するためのものである
「世界」とは「言語」を共有する人の集まりである

これらの世界観を実現するために、ある領域における関係性や定義を表現するツールを作りたい。
spec_detail.md のようなイメージをしている。
